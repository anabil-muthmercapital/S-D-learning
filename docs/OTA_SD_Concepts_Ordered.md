# مفاهيم Supply & Demand — مرتّبة من الصفر

> **الفكرة من الملف ده:** كل مفهوم مبني على اللي قبله. متعدّيش لمفهوم قبل ما تفهم اللي تحته. كل قسم فيه: **التعريف** (يعني إيه)، **المعادلة**، **مثال رقمي**، و**ليه موجود** (المنطق).

> **المثال اللي هنمشي عليه:** سهم افتراضي اسمه XYZ على فريم الساعة. الـ base شمعات C3-C4-C5، شمعة الخروج C6. و `ATR = 1.6`. هنستخدم نفس الأرقام في كل مكان عشان تشوف الترابط.

---

# المستوى صفر — اللبنة الأساسية

## 1. الشمعة (Candle)

### التعريف
كل شمعة بتمثّل فترة زمنية (ساعة، يوم، أسبوع...). وفيها **٤ أسعار**:
- **Open** — سعر الافتتاح (أول السعر في الفترة).
- **Close** — سعر الإغلاق (آخر السعر).
- **High** — أعلى سعر وصله.
- **Low** — أقل سعر وصله.

ومنهم بنشتق ٣ حاجات:
- **Body (الجسم)** = المسافة بين الافتتاح والإغلاق. ده "صافي الحركة".
- **Range (المدى)** = المسافة بين الأعلى والأقل. دي "كل الحركة بما فيها التذبذب".
- **Wick (الفتيل)** = الخطوط الرفيعة فوق وتحت الجسم. أسعار اتلمست بس السوق رفضها ورجع.

**اللون:** خضرا لو `close > open` (صعود)، حمرا لو `close < open` (هبوط).

### المعادلة
```
body  = |close − open|
range = high − low
```

### مثال رقمي
شمعة: Open=100، High=105، Low=99، Close=103.
```
body  = |103 − 100| = 3
range = 105 − 99    = 6
اللون: 103 > 100 → خضرا
```

### ليه موجود؟
دي اللبنة. كل المفاهيم بعد كده بتتعامل مع الأرقام الأربعة دي. لو مش فاهم الشمعة، مش هتفهم أي حاجة بعدها.

---

## 2. ATR — متوسط المدى الحقيقي

### التعريف
**ATR (Average True Range)** = متوسط حركة الشمعة على آخر ١٤ شمعة. بكلام بسيط: **قد إيه السعر بيتحرك في المتوسط في الشمعة الواحدة**. ده مقياس **التقلب (volatility)**.

ده "المسطرة" اللي بنقيس بيها كل حاجة بعد كده.

### المعادلة
أول حاجة، **المدى الحقيقي (True Range)** لكل شمعة — الأكبر من ٣ قيم:
```
TR = max(
    high − low,                  ← مدى الشمعة نفسها
    |high − close_السابق|,       ← القفزة من إغلاق أمس (gap لأعلى)
    |low  − close_السابق|        ← القفزة من إغلاق أمس (gap لأسفل)
)
```
وبعدين **المتوسط** على آخر ١٤ شمعة:
```
ATR = متوسط متحرك لـ TR على آخر 14 شمعة
```

### مثال رقمي
لو متوسط حركة XYZ طلع `1.6` → يعني السهم بيتحرك حوالي 1.6 نقطة في الساعة في المتوسط.

### ليه موجود؟ (أهم نقطة في الملف)
عشان نخلّي كل حساباتنا **نسبية** بدل مطلقة.
- حركة 3 نقاط على سهم ATR بتاعه 0.5 = حركة **ضخمة**.
- نفس الـ 3 نقاط على سهم ATR بتاعه 10 = حركة **تافهة**.

الـ ATR هو اللي بيخلّي المؤشر يشتغل على أي سهم وأي سعر من غير ما تغيّر الإعدادات. عشان كده هتلاقينا **بنقسم على ATR في كل مكان** — عشان نحوّل الأرقام لحكم له معنى.

# المستوى الأول — تصنيف الشمعة

## 3. Body Ratio — نسبة الجسم للمدى

### التعريف
أهم نسبة بنشتقها من الشمعة. بتفرّق الشمعة **الضعيفة** (base) عن **القوية** (leg).

### المعادلة
```
body_ratio = body / range = |close − open| / (high − low)
```
**القاعدة:**
- `body_ratio ≤ 0.5` → شمعة **ضعيفة** → جزء من **base**.
- `body_ratio` كبير (قريب من 1) → شمعة **قوية** → **leg**.

### مثال رقمي
| الشمعة | body | range | body_ratio | التصنيف |
|--------|------|-------|------------|---------|
| C3 | 0.2 | 0.7 | 0.29 | ضعيفة → Base |
| C4 | 0.1 | 0.6 | 0.17 | ضعيفة → Base |
| C5 | 0.2 | 0.5 | 0.40 | ضعيفة → Base |
| C6 | 3.6 | 3.9 | 0.92 | قوية → Leg |

### ليه موجود؟
- جسم صغير نسبة للمدى = **البائعين والمشترين متعادلين** = توازن = بيئة الـ base.
- جسم كبير نسبة للمدى = **طرف كسب الآخر بوضوح** = اختلال = شمعة خروج (leg).

**ليه النسبة مش حجم الجسم لوحده؟** عشان شمعة جسمها نقطتين ممكن تكون قوية على سهم هادي وضعيفة على سهم متقلب. النسبة بتلغي حجم السهم وتقولك: الجسم ده كبير ولا صغير **بالنسبة لحركة الشمعة نفسها**.

# المستوى الثاني — تجميع الشمعات

## 4. Base — منطقة التوازن

### التعريف
**Base** = مجموعة شمعات ضعيفة متراصة (body_ratio ≤ 0.5 لكلهم). دي منطقة **توازن** — السعر متردد، محدش كسب.

في مثالنا: C3, C4, C5 → **base من ٣ شمعات**.

### المعادلة (حدود الـ base)
```
base_high = max(highs of base candles)   ← أعلى فتيل
base_low  = min(lows  of base candles)    ← أقل فتيل
base_width = base_high − base_low
```

### مثال رقمي
```
base_high  = max(102.6, 102.4, 102.5) = 102.6
base_low   = min(101.9, 101.8, 102.0) = 101.8
base_width = 102.6 − 101.8 = 0.8
```

### ليه موجود؟
الـ base بيقولك **فين** المنطقة المحتملة. هنا المؤسسات بتجمّع أوامرها بهدوء قبل ما تحرّك السعر. بس لوحده مش كفاية — لازم نتأكد إنه ضيق (المفهوم الجاي) وإن الخروج منه قوي (لاحقاً).

> **ملاحظة عن الفتايل:** الـ base_high و base_low الافتراضي بيتحسبوا بالفتايل (high/low الكامل). فيه طريقة بديلة بتستخدم الأجسام (body) لو فيه فتيل شاذ — بس **اثبت على طريقة واحدة**، متخلطش.

---

## 5. Compactness — مدى ضيق الـ Base

### التعريف
الـ base الحقيقية لازم تكون **ضيقة أفقياً**. منطقة توازن حقيقية = السعر بالكاد بيتحرك. لو الـ base واسعة، ده مش توازن، ده تذبذب وهمي.

### المعادلة
```
compactness_ratio = base_width / ATR
الشرط: compactness_ratio ≤ 2.5
```
(كل ما الرقم أصغر، كل ما الـ base أضيق وأقوى)

### مثال رقمي
```
compactness_ratio = 0.8 / 1.6 = 0.5
الشرط: 0.5 ≤ 2.5 ✅ → ضيقة جداً وممتازة (نص الـ ATR بس)
```

### ليه موجود؟
عشان نمنع الـ "base الوهمية". شمعة ممكن body_ratio بتاعها صغير بس مداها ضخم → دي مش base، دي تذبذب عنيف. الـ compactness بيكشف ده.

> **فخ مهم:** متخلطش بين الـ compactness (عرض أفقي) والـ time (عدد الشمعات). دول مقياسين **مستقلين**: ممكن base من شمعتين عرضها واسع، وممكن base من ٥ شمعات عرضها ضيق.

---

## 6. Leg — شمعة الخروج

### التعريف
**Leg-out** = الشمعة/الشمعات القوية اللي خرجت من الـ base بعنف. دي اللي بتكسر التوازن.

في مثالنا: C6 (body_ratio = 0.92) → شمعة خروج قوية.

### القاعدة
```
الشمعة leg لو: body_ratio ≥ 0.6  (قوية)
واتجاهها بيحدد اتجاه المنطقة (الجاي)
```

### ليه موجود؟
الـ base لوحده توازن بلا معلومة. اللي بيحوّله لمنطقة "تستاهل" هو **قوة الخروج منه**. الـ leg هو الدليل إن حد ضخم (مؤسسة) كان مستني عند المنطقة ودخل بأوامر كبيرة.

---

## 7. الاتجاه والتشكيلات الأربعة

### التعريف
اتجاه المنطقة بيتحدد من **اتجاه الـ leg-out**:
- خرج لأعلى → **Demand** (طلب، نشتري).
- خرج لأسفل → **Supply** (عرض، نبيع).

### التشكيلات الأربعة
| النمط | الاسم | النوع | المعنى |
|-------|-------|-------|--------|
| RBR | Rally-Base-Rally | Demand | استمرار صاعد |
| DBD | Drop-Base-Drop | Supply | استمرار هابط |
| DBR | Drop-Base-Rally | Demand | انعكاس صاعد |
| RBD | Rally-Base-Drop | Supply | انعكاس هابط |

الحرف الأول = الدخول (leg-in)، الأوسط = Base، الأخير = الخروج (leg-out).

### مثال رقمي
```
Close(C6) = 105.6  >  base_high = 102.6  → خروج لأعلى ✅
الدخول كان صاعد (C1→C2) + الخروج صاعد (C6) → RBR (Demand)
```

### ليه موجود؟
عشان نعرف نشتري ولا نبيع، وعشان نفرّق بين **الاستمرار** (RBR/DBD) و**الانعكاس** (DBR/RBD).

> **فخ:** متعتمدش على لون شمعة الخروج بس. لازم الإغلاق **يتعدّى حدود الـ base فعلاً** (اختراق حقيقي، مش شمعة خضرا جوه النطاق).

# المستوى الثالث — تقييم المنطقة (الأهم)

## 8. Departure — قوة الخروج (المعيار الأساسي)

### التعريف
**Departure** معناها "المغادرة". هي حركة خروج السعر من الـ base **بعنف وسرعة**، مقيسة كرقم. دي **أهم معيار في المنهج كله**.

### المعادلة — قياسان لازم يتحققا
```
departure_distance = |close(leg) − base_high|        (لمنطقة الطلب)

القياس 1: departure_ratio = departure_distance / base_width
          الشرط: ≥ 2.0

القياس 2: departure_atr = departure_distance / ATR
          الشرط: ≥ 0.8
```

### مثال رقمي
```
departure_distance = |105.6 − 102.6| = 3.0

departure_ratio = 3.0 / 0.8 = 3.75   → ≥ 2.0 ✅✅ (أقوى من المطلوب)
departure_atr   = 3.0 / 1.6 = 1.875  → ≥ 0.8 ✅✅
```

### ليه قياسين مش واحد؟ (الفخ المهم)
كل قياس بيجاوب سؤال مختلف:
1. `departure_ratio` بيسأل: هل الخروج كبير **مقارنة بحجم الـ base**؟
2. `departure_atr` بيسأل: هل الخروج كبير **مقارنة بتقلب السوق الطبيعي**؟

**مثال على الخداع:** base ضيقة جداً (`base_width = 0.1`)، والخروج `0.5`.
- `departure_ratio = 0.5 / 0.1 = 5.0` → رقم ضخم! القياس الأول يقول "ممتاز!"
- بس لو `ATR = 2.0`: `departure_atr = 0.5 / 2.0 = 0.25` → أقل من 0.8 → القياس الثاني **يرفض**.

الـ base الضيقة خدعت القياس الأول، والـ ATR كشفها. **عشان كده لازم الاتنين.**

### ليه ده أهم معيار؟
الـ base بيقولك **فين** المنطقة. الـ Departure بيقولك **هل المنطقة تستاهل أصلاً**. خروج عنيف = مؤسسة دخلت بقوة. خروج ضعيف = منطقة بلا قيمة، السعر هيخترقها بسهولة لما يرجع.

## 9. Proximal / Distal — حدود المنطقة

### التعريف
- **Proximal** = الحد **الأقرب للسعر** لما يرجع للمنطقة = **نقطة الدخول**.
- **Distal** = الحد **الأبعد** (آخر دفاع) = **مكان الستوب**.

### القاعدة (احفظها بالمنطق)
proximal دايماً الأقرب للسعر لما يرجع، distal دايماً الأبعد.

| | Proximal (الدخول) | Distal (الستوب) |
|---|---|---|
| **Demand** (السعر نازل من فوق) | base_high (الحد العلوي) | base_low (الحد السفلي) |
| **Supply** (السعر طالع من تحت) | base_low (الحد السفلي) | base_high (الحد العلوي) |

### مثال رقمي (منطقة طلب)
```
proximal = base_high = 102.6   (الدخول)
distal   = base_low  = 101.8   (الستوب وراه)
zone_width = 102.6 − 101.8 = 0.8
```

### ليه موجود؟
عشان نعرف **فين ندخل وفين نحط الستوب**. الخطأ الأشهر في المنهج كله: عكس proximal/distal بين الطلب والعرض. القاعدة الثابتة تحميك: الأقرب للسعر وقت الرجوع = proximal.

# المستوى الرابع — جودة المنطقة

## 10. Freshness — الطزاجة

### التعريف
المنطقة "Fresh" لو السعر **لمسها مرات قليلة** بعد تكوينها. كل لمسة بتستهلك أوامر → بتضعّفها.

### المعادلة
```
touches = عدد الشمعات اللي close ≤ proximal (بعد التكوين)

freshness_score = 2  لو touches = 0   (طازة تماماً)
                = 1  لو touches = 1
                = 0  لو touches ≥ 2   (مستهلكة)
```

### مثال رقمي
في مثالنا، مفيش شمعة بعد C6 أغلقت ≤ 102.6 → `touches = 0` → **score = 2 (طازة)**.

### ليه موجود؟
أول مرة السعر يرجع للمنطقة، بيكون فيها أوامر كتير → ارتداد قوي. بعد لمستين-تلاتة، الأوامر بتخلص → المنطقة مستهلكة وضعيفة. وكمان: لو السعر **أغلق تحت الـ distal**، المنطقة **ماتت** (اتكسرت) وتتشال خالص.

---

## 11. Time — عدد شموع الـ Base

### التعريف
كل ما الـ base أقصر زمنياً (شمعات أقل)، كل ما التوازن أحسم وأقوى.

### المعادلة
```
T_score = 2  لو base_count ≤ 2
        = 1  لو base_count = 3
        = 0  لو base_count ≥ 4
الحد الأقصى: base_count ≤ 5
```

### مثال رقمي
```
base_count = 3 (C3,C4,C5) → T_score = 1
```

### ليه موجود؟
base قصيرة = اختلال حسم بسرعة = أقوى. base طويلة (٥+) ممكن تكون نطاق تجميع كبير مش base حقيقية.

---

## 12. The Curve — موقعك في النطاق

### التعريف
بنقسم النطاق الكبير (بين أكبر عرض وأكبر طلب على الفريم الأعلى) لـ ٣: Low / Mid / High. القاعدة: **اشترِ في Low، بِع في High**.

### المعادلة
```
position = (السعر − htf_demand_distal) / (htf_supply_distal − htf_demand_distal)

Low   لو position < 0.33   → نقطة 2 (للشراء)
Mid   لو 0.33–0.66          → نقطة 1
High  لو position > 0.66   → نقطة 2 (للبيع)
```

### مثال رقمي
```
position = (103.2 − 98) / (120 − 98) = 5.2 / 22 = 0.236
0.236 < 0.33 → Low Third → شراء رخيص ✅ → نقطة 2
```

### ليه موجود؟
عشان متشتريش غالي. منطقة طلب ممتازة في أعلى النطاق = بتشتري في القمة = خطر. نفس المنطقة في أسفل النطاق = صفقة ذهبية.

---

## 13. Trend Alignment — التوافق مع الاتجاه

### التعريف
المنطقة لازم تكون **مع الاتجاه**. بنكشف الاتجاه بطريقتين لازم يتفقوا:
- **M1 (هيكل):** قمم وقيعان صاعدة = up، هابطة = down.
- **M2 (كسر مناطق):** كسر عرض لأعلى = up، كسر طلب لأسفل = down.

### المعادلة
```
trend = M1            لو M1 == M2 (اتفقوا)
      = 0 (sideways)  لو اختلفوا

T2_score = 2  لو المنطقة مع الاتجاه
         = 1  لو sideways
         = 0  لو المنطقة عكس الاتجاه
```

### مثال رقمي
```
M1 = +1 (قمم وقيعان صاعدة) ، M2 = +1 (كسر عرض لأعلى)
trend = UP. منطقتنا Demand مع UP → T2_score = 2 ✅
```

### ليه موجود؟
منطقة شراء وسط ترند هابط = بتسبح ضد التيار. **الأفضل تمنع الصفقة عكس الاتجاه تماماً**، مش بس تضعّف تقييمها.

# المستوى الخامس — القرار

## 14. S.E.T.S — تجميع النقاط

### التعريف
بنجمع نقاط المعايير الخمسة في score واحد ونحوّله لنجوم.

### المعادلة
```
total_score = S(departure) + T(time) + F(freshness) + T(trend) + Curve

★★★  لو total_score ≥ 7
★★   لو 5–6
★    لو ≤ 4
```

### مثال رقمي
| المعيار | المصدر | النقطة |
|---------|--------|--------|
| S — Strength | departure_ratio = 3.75 | 2 |
| T — Time | base_count = 3 | 1 |
| F — Freshness | touches = 0 | 2 |
| T — Trend | demand مع uptrend | 2 |
| Curve | Low Third | 2 |
| **المجموع** | | **9/9 → ★★★** |

### ليه موجود؟
عشان قرار موضوعي بدل "حاسس إنها حلوة". (لاحقاً الـ ML ممكن يتعلّم الأوزان الحقيقية لكل معيار من البيانات بدل التخمين).

## 15. Nested Zones — التداخل

### التعريف
لو منطقة فريم صغير واقعة جوه منطقة فريم أكبر → إشارة أقوى. بس لازم نقيس **نسبة التداخل** الفعلية.

### المعادلة
```
overlap_ratio = overlap / zone_width
is_nested = overlap_ratio ≥ 0.5   (تداخل حقيقي ≥ 50%)
nesting_bonus = +1 نقطة
```

### ليه موجود؟
التداخل عبر الفريمات = اتفاق بين اللاعبين الكبار والصغار = ثقة أعلى. شرط الـ 50% بيمنع إن مجرد تلامس طرفي يتحسب nested غلط.

---

## 16. تنفيذ الصفقة — Entry / SL / TP

### التعريف
بعد ما المنطقة تعدّي كل المعايير، نحسب الصفقة الفعلية.

### المعادلة (منطقة طلب — شراء)
```
entry = proximal
sl    = distal − buffer        (buffer = 0.1 × ATR)
risk  = entry − sl
tp    = entry + 3 × risk        (RR = 3)
```

### مثال رقمي
```
entry = 102.6
buffer = 0.1 × 1.6 = 0.16
sl = 101.8 − 0.16 = 101.64
risk = 102.6 − 101.64 = 0.96
tp = 102.6 + 3 × 0.96 = 105.48
```
→ الصفقة: ادخل 102.6، ستوب 101.64، هدف 105.48 (1:3).

### ليه نسبة 1:3؟
```
نقطة التعادل = 1 / (1 + RR) = 1/4 = 25%
```
يعني تكفيك **٢٥٪ صفقات ناجحة** عشان متخسرش. أي حاجة فوق كده = ربح. ده بيخليك رابح حتى لو أغلب صفقاتك تخسر.

# المستوى السادس — السياق الأوسع

## 17. اختيار التايم فريم

استخدم **٣ أطر**، كل واحد أكبر من اللي تحته بـ 4–6 أضعاف:
- **HTF (الأعلى)** = الاتجاه (البوصلة).
- **ITF (الأوسط)** = التأكيد.
- **LTF (الأقل)** = الدخول (ساعة التوقيت).

**القاعدة:** متاخدش صفقة على الفريم الأقل إلا لو مع اتجاه الفريم الأعلى.

| الأسلوب | HTF | ITF | LTF |
|---------|-----|-----|-----|
| Intraday | 4h | 1h | 15m |
| Swing | أسبوعي | يومي | 4h |
| Position | شهري | أسبوعي | يومي |

## 18. السيولة وصيد الستوبات (Stop Hunt)

تحت كل منطقة طلب فيه تجمّع أوامر وقف خسارة = سيولة. المؤسسات بتنزل السعر بسرعة تخترق الـ distal بفتيل (تجمع السيولة)، وبعدين ترتد بعنف.

**عملياً:**
- متحطش الستوب عند الحافة بالظبط — حطه بعد buffer.
- فتيل كاذب تحت المنطقة + إغلاق فوقها = **إشارة قوة** (مش ضعف).
- فرق بين اختراق بفتيل (stop hunt = فرصة) واختراق بإغلاق (كسر حقيقي = موت المنطقة).

---

## 19. FVG — الفجوة السعرية (Fair Value Gap)

فراغ سعري بتسيبه حركة عنيفة سريعة (نفس فكرة الـ departure القوي). السوق بيميل يرجع يملأ الفجوة.

### المعادلة (نمط 3 شمعات صاعد)
```
الفجوة موجودة لو: Low(الشمعة الثالثة) > High(الشمعة الأولى)
```

**العلاقة بـ S&D:** الاختلال اللي بيكوّن الـ leg-out القوي هو نفسه اللي بيسيب الـ FVG. لو منطقتك فيها FVG = تأكيد إضافي على القوة. (مفيد كتأكيد، مش كإشارة منفصلة).

## 20. إدارة المخاطر (أهم قسم عملياً)

### القاعدة الأولى
لا تخاطر بأكتر من **1–2%** من رأس مالك في الصفقة الواحدة، مهما كانت ثقتك.

### المعادلة (حجم الصفقة)
```
المخاطرة بالدولار = رأس المال × نسبة المخاطرة
حجم الصفقة = المخاطرة بالدولار / عرض الستوب
حيث: عرض الستوب = |entry − sl|
```

### مثال رقمي
```
رأس المال = 10,000$ ، نسبة المخاطرة = 1%
المخاطرة بالدولار = 10,000 × 0.01 = 100$
عرض الستوب = 102.6 − 101.64 = 0.96
حجم الصفقة = 100 / 0.96 ≈ 104 وحدة
```
→ مهما كان عرض المنطقة، خسارتك القصوى تظل **100$ ثابتة**.

### ليه ده أهم من إشارات الدخول؟
- متداول بيخاطر 1% ممكن يخسر 20 صفقة متتالية ويفضل عنده 80% من حسابه.
- متداول بيخاطر 10% بيفلس بعد 7 صفقات خاسرة.

**البقاء في السوق أهم من الربح السريع. احترم المخاطرة قبل أي حاجة.**

# 🧮 جدول المعادلات السريع (مرجع)

| المفهوم | المعادلة |
|---------|----------|
| Body ratio | `\|close−open\| / (high−low)` |
| Base width | `max(base highs) − min(base lows)` |
| Compactness | `base_width / ATR ≤ 2.5` |
| Departure ratio | `\|close_leg − proximal\| / base_width ≥ 2.0` |
| Departure ATR | `\|close_leg − proximal\| / ATR ≥ 0.8` |
| Zone width | `proximal − distal` |
| Freshness | `count(close ≤ proximal بعد التكوين)` |
| Curve position | `(price − dem_distal) / (sup_distal − dem_distal)` |
| Overlap ratio | `overlap / zone_width ≥ 0.5` |
| Risk | `entry − sl` |
| Take Profit | `entry + 3 × risk` |
| Break-even win% | `1 / (1 + RR)` |
| Position size | `(capital × risk%) / stop_width` |

# 📌 ملخّص ترتيب التعلّم

```
المستوى صفر: الشمعة → ATR              (اللبنة + المسطرة)
المستوى 1:   Body Ratio                (تصنيف الشمعة)
المستوى 2:   Base → Compactness → Leg → التشكيلات   (تجميع)
المستوى 3:   Departure → Proximal/Distal            (التقييم الأساسي)

```
